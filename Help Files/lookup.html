Salesforce objects often include lookup fields that allow users to associate two records together in a relationship. For example, a contact record includes an Account lookup field that represents the relationship between the contact and its associated organization. Lookup fields appear with the Lookup button on record edit pages. Clicking Lookup opens a lookup search dialog that allows you to search for the record that should be associated with the one being edited. When searching for records with the lookup screen, the scope of search is limited to fields configured by your administrator.
Enter your search keyword here and tap the Search button on keyboard. Tap here to view the record. Tap here to see the list of Actions that can be performed on this record. Tap on any record to select it for the lookup field. Tap here to select the controlling picklist value from the pop up. Tap here to select the dependent picklist value from the pop up.
Rules
Fields to search against, fields to display, and number of records in the result set are configured by your administrator in SFM Transaction Designer.
A dependent picklist works in conjuction with a controlling picklist to filter the available options. The value chosen in the controlling field affects the values available in the dependent field. The controlling field can be a checkbox or a picklist.
Lookup window title shows the object's label and the field label to provide a better context to the search.
Save and Quick Save under Actions are enabled based on the configuration in the SFM transaction designer.
