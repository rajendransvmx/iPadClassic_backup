This document outlines the setup activities in ServiceMax/Salesforce that drive the functionality and user experience of the ServiceMax iPad client. The functionality in the iPad application will work as intended only if the setup is completed as instructed below. Familiarity with Salesforce and ServiceMax setup is needed to successfully perform the steps.
Device requirements
ServiceMax iPad client will work with all models of iPad. The minimum configuration is iOS v4.2. Internet connectivity is required, either through 3G or Wi-Fi. If there is a proxy or firewall in your network, the iPad must be configured to get the appropriate access to salesforce.com.
Synchronization
Events from the current day and week are cached locally for better performance. The cache can be refreshed manually by tapping on the Refresh button in the Day View of the calendar or by navigating to a day in a different week. The cache is local to the device and the user.
Updates made from iPad client to work orders and other objects are saved to Salesforce immediately and synchronized back to the local database if there is online connectivity.
Dataset synchronization will happen as per the mobile configuration in Salesforce.
The application will continue to work offline with the local dataset if there is no online connectivity.
Product
To view the product’s picture on the chatter screen, create an image attachment to the Product record in Salesforce. The attachment file name should contain the word PICTURE in it. For example: CS2000 PICTURE.jpg or PICTURE OF CS2000.jpg. Supported valid image formats are jpeg, png, gif and bmp. Using unsupported image formats can result in unpredictable behavior in the application.
To view product manuals in the iPad client, create a PDF attachment with the word MANUAL in the file name. For example: CS2000 MANUAL.pdf or MANUAL for CS2000.pdf
The product manuals are downloaded/cached when there is online connectivity. Downloaded/cached copy will be used in subsequent invocations.
Chatter
To collaborate using chatter on iPad, enable chatter in Salesforce and specifically for the product object. Refer to Salesforce online help for more information on enabling chatter.
For adding your photo to the product chatter feed on the application, perform the following steps on Salesforce.
  1. Create a new folder under Documents called ServiceMax for iPad. Choose the This folder is accessible by all users setting.
  2. Create a new document with name as your fully qualified user name (e.g., yourlogin@yourcompany.com).
  3. Upload your photo in PNG format. Note that only PNG format is supported.
  4. Save it in the folder created in Step 1.
Contact
To view the contact’s picture on the job details screen, create an image attachment to the Contact record in Salesforce. The attachment file name should contain the word PICTURE in it. For example: John Doe PICTURE.jpg or PICTURE OF John Doe.jpg. Supported valid image formats are jpeg, png, gif and bmp. Using unsupported image formats can result in unpredictable behavior in the application.
Knowledgebase Articles for troubleshooting
For creating knowledge base articles, perform the following steps:
  1. Create a folder whose name exactly matches the title of the KB article.
  2. Create index.html page in which hyperlinks, images and video files can be embedded in the folder. The hyperlinks can be relative local paths or web-based URLs.
  3. Zip the folder. For optimum performance, make sure the zip file size does not exceed 3 MB.
  4. Login to Salesforce and navigate to Documents tab.
  5. If a dedicated document folder does not exist for KB articles, create a folder first.
  6. Create a new document as follows:
    1. Document Name = Title of the KB article.
    2. Folder = Name of the folder created in step #5.
    3. Description = Detailed description of this solution.
    4. Keywords = Comma-separated list of keywords that must contain the exact product name, can be used to locate this document. Typically product name, family, problem type etc..
    5. Choose the Zip file created in step #3.
  7. Click Save.
Service Report
To change the logo on the Service Report, perform the following steps in Salesforce.
  1. Create an image file with the logo as follows:
    1. Name = ServiceMax_iPad_CompanyLogo
    2. Format = PNG only
    3. Maximum dimensions allowed = 600 (width) x 250 (height)
    4. Minimum resolution required = 72 dpi
  2. Upload the file to the ServiceMax for IPad folder under Documents. Create the folder if it doesn't exist already.
