Setup Requirements
This screen presents a Salesforce record with its child records in Create or Edit mode, as configured by your administrator using SFM Transaction Designer. Depending upon the business process context, same type of record may be edited using multiple screens, to capture information on fields different from each other.
Tap here to view all header sections. Tap the header section name to see information from the section only. Tap the child section name to add/edit records for the section. Tap on the lookup field to search and select record from the popup. Tap on the multi-picklist field to select one or more values from the popup. Tap on the dotted label to see full label. Tap here to search and add multiple records to the child section. Tap here to add one record to the child section. Swipe to view the DELETE button and delete the record. Tap here to view or edit the child record with all configured fields.
Rules
The layout, section and fields in the Create/Edit screens are fully configurable using SFM Transaction Designer.
You can use only those SFM transactions for which your administrator has given access using SFM Wizard.
Header and line records are saved to Salesforce only when you choose to save. Line records are not saved intermediately.
Field default values are populated as configured in the SFM Transaction.
Custom SFM transactions can be launched only in View mode.
Navigation
Shows relevant standard actions and any custom page-level events configured by your administrator. Standard actions include Quick Save, Save, and Cancel.
Allows you to capture an approval signature on iPad and save it as an attachment to this record in Salesforce. Signature is saved only when you save the record.
Takes you to the screen from which you opened this record for creating or editing.
Indicates synchronization completed successfully.
